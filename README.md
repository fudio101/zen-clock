# ZenClock

A beautiful clock project running on the **LilyGo T-Display-S3** board.

## Hardware

| Spec | Value |
|---|---|
| **Board** | LilyGo T-Display-S3 (non-touch, v1.2) |
| **MCU** | ESP32-S3R8 (Dual-core Xtensa LX7, 240 MHz) |
| **Flash** | 16 MB |
| **PSRAM** | 8 MB Octal (OPI) |
| **Display** | 1.9" ST7789, 320×170, Intel 8080 8-bit parallel |
| **Backlight** | PWM controlled via ESP-IDF LEDC (lcd_backlight component) |
| **Battery** | GPIO4 ADC via resistor divider |
| **Buttons** | GPIO0 (BOOT) + GPIO14 — brightness control |

## Software Stack

| Layer | Technology |
|---|---|
| **Framework** | ESP-IDF v6.0.0 (via PlatformIO) |
| **Graphics** | LVGL 9.x |
| **LVGL Port** | [esp_lvgl_port](https://github.com/espressif/esp-bsp/tree/master/components/esp_lvgl_port) |
| **UI Design** | [SquareLine Studio](https://squareline.io/) |
| **Build System** | PlatformIO + ESP-IDF Component Manager |

## Getting Started

### Prerequisites

1. [PlatformIO](https://platformio.org/) installed (VS Code extension recommended)
2. [SquareLine Studio](https://squareline.io/downloads) v2.x installed (for UI design)
3. LilyGo T-Display-S3 connected via USB

### Build & Flash

```bash
# Build the project
pio run

# Flash to device
pio run -t upload

# Monitor serial output
pio device monitor
```

### First Boot

After flashing, the display shows the SquareLine-designed UI with a smooth 2-second backlight fade-in. Use the buttons to adjust brightness:
- **BOOT button** (GPIO0) — Brightness UP (+10%)
- **Side button** (GPIO14) — Brightness DOWN (-10%)

## SquareLine Studio Setup (Step-by-Step)

SquareLine Studio lets you design the clock UI visually (drag & drop) and automatically generates LVGL C code.

### Step 1: Install the Board Template

Copy the ZenClock board template into SquareLine Studio's board directory:

**Windows:**
```
Copy the folder:
  ZenClock/SquareLine/boards/t_display_s3/

To:
  C:/Users/<YourUser>/SquarelineLogs/boards/espressif/
```

> **Tip:** The exact path depends on your SquareLine Studio installation. Check `File → Preferences → Board Path` in SquareLine Studio to find where it looks for board templates.

### Step 2: Create a New Project

1. Open **SquareLine Studio**
2. Click **Create** → **New Project**
3. In the board selector, look under the **LilyGO** group
4. Select **"ZenClock (T-Display S3)"**
5. Verify the settings:
   - Resolution: **320 × 170**
   - Color depth: **16-bit (swapped)**
   - LVGL version: **9.x**
   - Language: **C**
6. Click **Create**

### Step 3: Design Your UI

- Drag and drop widgets (labels, arcs, images, etc.) onto the 320×170 canvas
- The canvas represents the physical screen in **landscape** orientation (buttons on the left side)
- Use LVGL's built-in fonts (Montserrat 12/14/20/24 are enabled in the config)
- Save your project (`.spj` file) inside `ZenClock/SquareLine/` for easy access

### Step 4: Export UI Code

1. Go to **File → Project Settings**
2. Verify **UI Files Export Path** points to `./components/ui`
   - This is relative to the ZenClock project root
   - If SquareLine uses absolute paths, set it to `D:/esp32/ZenClock/components/ui`
3. Click **Export → Export UI Files**
4. The `components/ui/` folder will be populated with:
   - `ui.h` / `ui.c` — Main UI initialization
   - `ui_helpers.h` / `ui_helpers.c` — Animation and event helpers
   - Screen files (e.g., `screens/ui_Screen1.c`)
   - Asset files (fonts, images)
   - `CMakeLists.txt` — Auto-generated build config (**must be patched!**)

### Step 4.5: Patch CMakeLists.txt (Required!)

SquareLine generates a plain `add_library()` CMakeLists.txt that is **incompatible** with ESP-IDF's component system. You must patch it after every export:

```bash
# From the project root
patch_ui.bat
```

This replaces the generated `add_library(ui ...)` with `idf_component_register(...)` using `GLOB_RECURSE` to auto-discover all source files.

> ⚠️ **Forgetting this step will cause linker errors** (`undefined reference to ui_init`) or display artifacts.

### Step 5: Integrate into main.c

The project is already configured to use `ui_init()`. The current `src/main.c`:

```c
#include "bsp.h"
#include "ui.h"

void app_main(void)
{
    static lv_display_t *disp_handle;
    bsp_display_init(&disp_handle, false);

    lvgl_port_lock(0);
    ui_init();
    lvgl_port_unlock();

    bsp_display_set_brightness(100, 2000);
    bsp_buttons_init(on_button_press);
    // app_main returns — FreeRTOS scheduler continues
}
```

> **Note:** `ui_init()` creates all screens, loads the default theme, and displays `Screen1`. You can access any widget exported by SquareLine (e.g., `ui_Label1`, `ui_Screen1`) via the `ui.h` header.

### Step 6: Build & Flash

```bash
pio run -t upload && pio device monitor
```

Your designed UI should now appear on the display! 🎉

### Re-exporting After Changes

Every time you modify the UI in SquareLine Studio:
1. Click **Export → Export UI Files** (this overwrites `components/ui/`)
2. Run `patch_ui.bat` to fix CMakeLists.txt
3. Rebuild: `pio run -t upload`

> **Important:** Do NOT manually edit files inside `components/ui/`. They will be overwritten on the next export. Put custom logic in `src/main.c` or create a separate component.

## Menuconfig (Required Settings)

After cloning or setting up the project for the first time, run:

```bash
pio run -t menuconfig
```

Ensure the following settings are configured:

| Setting | Path | Value |
|---|---|---|
| PSRAM Mode | Component config → ESP PSRAM → SPI RAM config → Mode | **Octal** |
| PSRAM Speed | Component config → ESP PSRAM → SPI RAM config → Speed | **80 MHz** |
| CPU Frequency | Component config → ESP System Settings → CPU frequency | **240 MHz** |
| FreeRTOS Tick | Component config → FreeRTOS → Kernel → Tick rate (Hz) | **1000** |
| Flash Size | Serial flasher config → Flash size | **16 MB** |
| LVGL Color Depth | Component config → LVGL → Display → Color depth | **16** |
| Montserrat 14 | Component config → LVGL → Font usage → Enable Montserrat 14 | **✓** |
| Montserrat 20 | Component config → LVGL → Font usage → Enable Montserrat 20 | **✓** |

> ⚠️ **Never set PSRAM to Quad mode** — this will cause a boot loop on the T-Display-S3.

## Known Gotchas & Troubleshooting

### Display Garbled / Smeared

The ST7789 display orientation must be configured in **two places** with matching values:

1. **Panel hardware** (`bsp_display.c` → `init_panel()`):
   ```c
   esp_lcd_panel_swap_xy(panel_handle, true);
   esp_lcd_panel_mirror(panel_handle, false, true);
   ```
2. **LVGL port** (`bsp_display.c` → `register_lvgl_display()`):
   ```c
   .rotation = {
       .swap_xy = true,
       .mirror_x = false,
       .mirror_y = true,
   },
   ```

> If these don't match, or if only one is set, the display will appear garbled.

### Build Fails After UI Export

SquareLine Studio generates a plain `add_library()` CMakeLists.txt. Always run `patch_ui.bat` after exporting to convert it to `idf_component_register()`.
